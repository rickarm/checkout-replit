REPLIT PROMPT

Build a clean v1 web app for my journaling tool called Checkout.

Context:
- Checkout already exists today as a Node.js CLI app that stores journal entries as markdown files.
- A separate Phase 1 refactor is underway to create a shared service layer and storage abstraction.
- For now, do NOT depend on that refactor being complete.
- Instead, build the web app as a standalone shell using mocked data and clear integration seams.
- This app should eventually support multiple storage backends, especially Google Drive.
- The product philosophy is calm, structured reflection with portable user-owned data.

Primary goal:
Create a polished web app shell that I can click through and evaluate for UX and product direction, even before real storage is wired up.

Requirements:
1. Build a modern full-stack web app with a clean UI that works on desktop, tablet, and mobile interfaces
2. Use mocked journal data for now
3. Create obvious places where real services/storage can later plug in
4. Do not build a mobile app
5. Do not build Google Drive integration yet, but prepare the app for it
6. Do not use a database as the canonical source of truth for journal entries
7. Keep the design calm, minimal, and reflective rather than busy or “productivity app” feeling

Screens to build:
1. New Entry
2. Journal List
3. Entry Detail / Edit
4. Settings / Storage

Navigation:
- Simple top nav or left nav with:
  - New Entry
  - Journal
  - Settings

UI requirements:
- Calm, spacious layout
- Excellent readability
- Feels good on desktop and iPad-sized screens
- Minimal but polished visual design
- Avoid clutter
- Journal writing experience should feel focused and grounded

Functional requirements for v1:

  - Show today’s date
  - Show a small set of guided prompts; see below for specific prompts
  - Include text areas for each prompt
  - Save draft button
  - Save entry button
- Journal List screen:
  - Reverse chronological list of entries
  - Group or filter by month
  - Search input stub is fine, even if mock-only
  - Clicking an entry opens detail view
- Entry Detail / Edit screen:
  - Show entry contents in a readable layout
  - Allow editing prompt answers
  - Allow viewing rendered markdown or structured content
- Settings screen:

    - Local files

2. What was one moment of joy?

4. Think of your values. One thing you did that aligns with a value?
5. What are letting go of? What is not serving you anymore?

- Do not design the app around a proprietary database model
- The storage abstraction should feel central to the product
- This app will eventually support both power users and normal users
